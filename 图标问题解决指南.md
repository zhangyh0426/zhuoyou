## 🔧 详细解决步骤

### 1. 清理微信开发者工具缓存
```
操作步骤：
1. 打开微信开发者工具
2. 点击顶部菜单：工具 → 编译
3. 在编译设置中勾选"清除缓存"
4. 或者：工具 → 清缓存 → 全部清除
5. 重新编译项目
```

### 2. 验证图标文件格式
检查您的图标文件是否符合要求：
- **格式要求**：PNG格式（推荐）
- **大小要求**：81px × 81px（推荐）
- **文件大小**：建议小于40KB
- **命名规范**：避免特殊字符和空格

### 3. 检查文件扩展名大小写
确保文件名完全一致：
```bash
# 正确示例
home.png          # 全部小写
home-active.png   # 全部小写，连字符连接
member.png        # 全部小写
member-active.png # 全部小写，连字符连接
activity.png      # 全部小写
activity-active.png # 全部小写，连字符连接

# 错误示例（需避免）
Home.png          # 首字母大写
HOME.PNG          # 全部大写，扩展名大写
home_active.png   # 下划线而非连字符
```

### 4. 项目配置文件检查
确认 `project.config.json` 配置正确：
```json
{
  "appid": "your_appid_here",
  "projectname": "狼人杀桌游店",
  "miniprogramRoot": "./",
  "cloudfunctionRoot": "cloudfunctions/",
  "compileType": "miniprogram"
}
```

### 5. 检查图标文件路径引用
在 `app.json` 中确认路径格式：
```json
{
  "tabBar": {
    "list": [
      {
        "pagePath": "pages/index/index",
        "text": "首页",
        "iconPath": "images/home.png",        // 相对路径
        "selectedIconPath": "images/home-active.png"  // 相对路径
      }
    ]
  }
}
```

## 🚨 常见错误及解决方案

### 错误1：404错误 - 资源文件未找到
**错误信息**：
```
Failed to load resource: the server responded with a status of 404 (Not Found)
```

**解决方案**：
1. 确认文件存在于正确目录
2. 检查路径大小写是否匹配
3. 清理缓存后重新编译

### 错误2：图标显示异常
**现象**：图标显示为灰色方块或默认图标

**解决方案**：
1. 检查图片文件是否损坏
2. 重新生成符合规范的图标文件
3. 确保图片透明背景

### 错误3：开发工具与真机显示不一致
**现象**：开发工具正常，真机不显示图标

**解决方案**：
1. 使用符合微信规范的图标尺寸
2. 确保网络连接稳定
3. 在真机上清除小程序缓存

## 📐 推荐图标规格

### TabBar图标标准
- **尺寸**：81px × 81px
- **格式**：PNG（推荐）
- **颜色**：单色或双色
- **风格**：简洁、辨识度高
- **文件大小**：≤ 40KB

### 图标设计建议
1. **保持简洁**：避免过于复杂的细节
2. **色彩统一**：与品牌色彩协调
3. **清晰可辨**：在小尺寸下依然清晰
4. **风格一致**：所有图标保持相同风格

```

2. 选择"在资源管理器中显示"
3. 确认文件路径和存在性
```
